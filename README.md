# autoagents5

一个面向 **多角色 AI 协作开发** 的提示词脚手架。目标是让 PM 产出结构化到"编号级"的 PRD 目录,使下游 UI / 架构师 / 前端 / 后端 / 测试 各 AI 角色**零联想**地完成交付。

## 目录

```
.
├── product/        原始英文 Agent 文档(5 个通用产品智能体)
├── productzh/      product/ 的中文镜像(一对一翻译)
├── agents/         ⭐ 多角色 AI 提示词脚手架(本仓库核心)
│   ├── PM/         产品经理 —— PRD 拆分与目录生成(重点)
│   ├── UI/         UI/UX 设计师
│   ├── Architect/  系统架构师
│   ├── Frontend/   前端工程师
│   ├── Backend/    后端工程师
│   └── QA/         测试工程师
└── docs/           各角色产出的唯一落盘位置(PRD / 设计 / 架构 / 测试)
```

## 快速开始

1. 阅读 [`agents/README.md`](./agents/README.md) 了解角色协作流水线与 GitHub 协作流程。
2. 以 **PM 角色**启动新项目:使用 [`agents/PM/产品经理-PRD拆分提示词.md`](./agents/PM/产品经理-PRD拆分提示词.md),让 AI 先生成 `docs/prd/` 骨架。
3. 并行启动 **UI** 与 **Architect**,产出设计与接口契约。
4. **Frontend / Backend** 按页面 ID 与接口 ID 并行开发;**QA** 对照 `40-acceptance.md` 驱动测试。

## 设计理念

- **PM 不决定技术栈**,只描述产品行为、三端布局、字段语义、业务规则、边界异常、埋点与验收。
- **每个子功能独立成文件夹**(如"忘记密码"、"记住密码"、"账号锁定" 各自独立),按统一模板书写。
- **编号即契约**:页面、字段、事件、错误码、实体、验收标准,全部有稳定编号,跨角色引用。
- **回写闭环**:下游发现歧义必须回写 `open-questions.md` + Issue,不允许私下脑补。

更多细节见 [`agents/PM/产品经理-PRD拆分提示词.md`](./agents/PM/产品经理-PRD拆分提示词.md)。
